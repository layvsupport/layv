# AISpace - 앱 작동 방식 문서

## 📌 프로젝트 개요
**AISpace**는 Electron 기반의 통합 AI 컨트롤 센터입니다.
여러 AI 서비스(ChatGPT, Claude, Gemini 등)를 하나의 데스크톱 앱에서 관리하고 제어할 수 있습니다.

**버전:** 1.0.0
**개발사:** BlendBytes
**기술 스택:** Electron 30.0.0, Vanilla JavaScript, CSS3

---

## 🏗️ 아키텍처 구조

### 1. Electron 메인 프로세스 (electron/main.js)
- **역할:** 앱 생명주기 관리, 창 생성, 네이티브 API 접근
- **주요 설정:**
  - 창 크기: 1600x960 (최소 1280x800)
  - webview 태그 활성화 (AI 사이트 임베딩)
  - contextIsolation 활성화 (보안)
  - sandbox: false (webview 사용 필요)

### 2. Preload 스크립트 (electron/preload.js)
- **역할:** 렌더러 프로세스와 메인 프로세스 간 안전한 통신
- **노출 API:**
  - `window.aispace.openExternalAI(url)` - 외부 AI 링크 열기

### 3. 렌더러 프로세스 (src/js/app.js)
- **역할:** UI 로직, 상태 관리, 사용자 인터랙션 처리
- **라인 수:** 1,322줄
- **주요 모듈:**
  - 캔버스 관리
  - 윈도우 관리
  - 드래그 앤 드롭
  - 프롬프트 브로드캐스팅
  - 테마 관리
  - 로컬 스토리지

## 🎯 핵심 기능

### 1. 멀티 캔버스 시스템
**구조:**
```
state.canvases = Map {
  'canvas-1': {
    id: 'canvas-1',
    name: 'Unified AI Canvas',
    windows: Map {
      'chatgpt-1234567890': { id, appId, app, promptEnabled },
      'claude-9876543210': { id, appId, app, promptEnabled }
    }
  },
  'canvas-2': { ... }
}
```

**특징:**
- 최대 6개 캔버스 생성 가능 (`MAX_CANVASES = 6`)
- 각 캔버스는 독립적인 AI 창 세트 보유
- 캔버스 이름 변경 가능 (더블클릭)
- 빈 캔버스 자동 삭제
- localStorage에 자동 저장/복원

**제한사항:**
- 캔버스당 최대 4개 AI 창 (`MAX_WINDOWS = 4`)
- 마지막 캔버스는 삭제 불가

### 2. AI 앱 카탈로그
**분류:** 5개 카테고리, 총 21개 AI 서비스

**카테고리:**
1. **대화형 AI** (4개)
   - ChatGPT, Claude, Gemini, Pi

2. **이미지 & 비디오** (4개)
   - Midjourney, DALL·E, RunwayML, Pika Labs

3. **리서치 & 문서** (4개)
   - Perplexity, Notion AI, Elicit, Mem

4. **코딩 어시스턴트** (4개)
   - GitHub Copilot, Cursor, Replit Ghostwriter, CodeWhisperer

5. **음성 & 음악** (3개)
   - Suno, ElevenLabs, Voiceflow

**앱 데이터 구조:**
```javascript
{
  id: 'chatgpt',
  name: 'ChatGPT',
  url: 'https://chat.openai.com/',
  icon: 'CG',
  accent: 'linear-gradient(...)',
  categoryId: 'conversation'
}
```

### 3. 빠른 실행 명령어
**입력 위치:** 사이드바 "빠른 실행" 섹션

**명령어 형식:** `/명령어` + Enter

**주요 명령어:**
```
/c, /chat, /chatgpt    → ChatGPT
/cl, /claude          → Claude
/g, /gem, /gemini     → Gemini
/p, /pi               → Pi
/m, /mj, /midjourney  → Midjourney
/d, /dalle            → DALL·E
/px, /perplexity      → Perplexity
/n, /notion           → Notion AI
```

**동작 방식:**
1. 입력창에 명령어 입력
2. Enter 키 누름
3. 현재 캔버스에 해당 AI 창 추가
4. 중복 체크 (이미 열린 경우 알림)
5. 피드백 메시지 표시 (2.4초)

### 4. 일괄 프롬프트 전송 (Broadcast)
**위치:** 사이드바 하단 "일괄 프롬프트 전송" 섹션

**작동 원리:**
1. 텍스트 영역에 프롬프트 입력
2. "전송" 버튼 클릭 또는 `Ctrl+Enter`
3. 현재 캔버스의 모든 "ON" 상태 창에 프롬프트 전송
4. 각 AI 사이트의 입력 필드에 텍스트 삽입

**지원 입력 선택자 (14개):**
```javascript
// ChatGPT
'textarea[placeholder*="Message"]'
'#prompt-textarea'

// Claude
'div[contenteditable="true"].ProseMirror'

// Gemini
'.ql-editor[contenteditable="true"]'

// 일반
'textarea', 'input[type="text"]', 'div[contenteditable="true"]'
```

**전송 메커니즘:**
- webview에 JavaScript 코드 주입 (`executeJavaScript`)
- 입력 필드 탐색 및 텍스트 설정
- React/Vue 감지를 위한 이벤트 발생:
  - `input` 이벤트
  - `change` 이벤트
  - `InputEvent` (inputType: 'insertText')

**창별 토글:**
- 각 AI 창 헤더에 ON/OFF 토글 스위치
- OFF 상태 창은 브로드캐스트에서 제외
- 상태는 localStorage에 저장

### 5. 드래그 앤 드롭
**작동 방식:**
1. 사이드바 AI 앱 아이콘을 드래그
2. 워크스페이스 캔버스에 드롭
3. 새 AI 창 생성

**데이터 전송:**
```javascript
event.dataTransfer.setData('application/json', JSON.stringify(app))
```

**상태 관리:**
```javascript
state.draggedApp = app  // 드래그 시작
state.draggedApp = null // 드래그 종료
```

**클릭 대안:**
- 앱 아이콘 클릭만으로도 창 추가 가능

### 6. 테마 시스템
**테마 종류:**
- **Light 모드:** 파스텔 그라데이션 (기본)
- **Dark 모드:** 다크 배경

**저장 위치:**
- localStorage: `aispace-theme`

**전환 방법:**
- 상단 바의 테마 토글 스위치
- 시스템 테마 자동 감지 (prefers-color-scheme)

**적용 방식:**
```javascript
document.documentElement.setAttribute('data-theme', 'dark')
```

### 7. 윈도우 리사이즈
**기능:**
- 각 AI 창 우측 하단 리사이즈 핸들
- 마우스 드래그로 크기 조절

**최소 크기:**
- 최소 너비: 240px
- 최소 높이: 220px

**구현:**
- Pointer Events 사용
- CSS `resize` 속성 + JavaScript 보조

### 8. 로컬 스토리지 영속성
**저장 데이터:**
1. **캔버스 상태** (`aispace-canvases`):
   - 모든 캔버스 정보
   - 각 캔버스의 윈도우 목록
   - 활성 캔버스 ID
   - 캔버스 ID 카운터

2. **테마 설정** (`aispace-theme`):
   - `light` 또는 `dark`

**저장 타이밍:**
- 캔버스 추가/삭제 시
- 윈도우 추가/삭제 시
- 캔버스 이름 변경 시
- 캔버스 전환 시
- 프롬프트 토글 변경 시

**복원:**
- 앱 시작 시 자동 복원
- 실패 시 기본 캔버스 생성

   ├─ getActiveCanvas()

   │   ├─ sendPromptToWebview(webview, text)

1. 사용자가 캔버스 탭 클릭
2. switchCanvas(canvasId)
   ├─ state.activeCanvasId = canvasId
   ├─ renderWorkspaceWindows()

   │       └─ createWindowElement()

## 🔐 보안 고려사항

### 1. Context Isolation
- **활성화 상태:** `contextIsolation: true`
- **목적:** 렌더러 프로세스와 preload 스크립트 분리
- **결과:** `window.aispace`로만 통신 가능

### 2. Sandbox
- **비활성화 이유:** `sandbox: false`
- **이유:** webview 태그 사용 필요
- **주의:** 보안 위험 존재 (신뢰하지 않는 코드 실행 금지)

### 3. Webview 격리
- 각 AI 사이트는 독립된 webview에서 실행
- 사이트 간 데이터 공유 불가
- 프롬프트 전송만 JavaScript 주입 사용

### 4. CSP (Content Security Policy)
- 현재 명시적 CSP 없음
- 개선 권장: meta 태그 또는 HTTP 헤더로 CSP 추가

## 🐛 알려진 제한사항

### 1. iframe vs webview
- **브라우저 버전:** iframe 사용 (CORS 제한)
- **Electron 버전:** webview 사용 (완전한 격리)
- **결과:** 브라우저에서는 일부 AI 사이트 로드 실패 가능

### 2. 프롬프트 전송 실패
- AI 사이트 구조 변경 시 선택자 업데이트 필요
- 페이지 로딩 완료 전 전송 시 실패
- 일부 사이트는 보안 정책으로 차단 가능

### 3. 성능
- 4개 AI 창 동시 실행 시 메모리 사용량 증가
- 각 webview는 독립 Chromium 프로세스
- 권장: 최대 2-3개 창 동시 사용

### 4. 크로스 플랫폼
- macOS: 테스트 필요
- Linux: 테스트 필요
- 일부 단축키는 플랫폼별 조정 필요

## 📝 개발 가이드라인

### 코드 스타일
- **언어:** Vanilla JavaScript (ES6+)
- **모듈:** 단일 파일 (app.js)
- **상태 관리:** 전역 `state` 객체
- **DOM 조작:** 직접 조작 (프레임워크 없음)

### 네이밍 컨벤션
- **함수:** camelCase (예: `createCanvas`)
- **상수:** SCREAMING_SNAKE_CASE (예: `MAX_WINDOWS`)
- **CSS 클래스:** BEM 변형 (예: `sidebar__app`)

### 이벤트 처리
- 이벤트 위임 사용 (sidebarList, canvasTabsList)
- 동적 요소는 data 속성 활용
- 메모리 누수 방지 위해 removeEventListener

### 로컬스토리지
- 키 접두사: `aispace-`
- JSON 직렬화/역직렬화
- try-catch로 에러 처리
